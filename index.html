Quick-n-dirty geo-location to city resolution

This service quickly resolves the nearest city/state for any latitude/longitude within the US. This service can quickly be expanded to outside countries, all that's needed is a lat/lon resolution table.

This was created on a challenge to port geo2city to Python. I accepted the challenge and ported it to both Python and AppEngine-my first attempt at a GAE app.

Example

http://geo2street.appspot.com/?loc=44.896352,-93.493097

Results

Parameters:

  • loc: can include multiple parameters, using [], in the format of 44.896352,-93.493097
  • format: xml for XML, json for JSON, human to be human readable
You can get a city-to-latitude file stored in city/state/lat/long format from http://sourceforge.net/projects/zips/.
